Partie 3 - Gestionnaire de version

Git et GitHub: définitions et différences

Un gestionnaire de version est un programme qui permet aux déveleppeurs de conserverun historique des modifications et des versions de tous leurs fichiers. Ainsi, il est possible de garder en mémoire chaque modification de chaque fichier.

3 grandes fonctionnalités: revenir à une version précédente en cas de problème, suivre l'évolution du code étape par étape, et travailler à plusieurs sans risquer de supprimer les modifications des autres collaborateurs.

Git est évidemment un gestionnaire de version. Il est utilisé pour créer un dépôt local et gérer les versions de vos fichiers. Tandis que GitHub est un service en ligne qui héberge des dépôts, il s'agit donc d'un dépôt distant puisqu'il n'est pas stocké sur un ordinateur personnel.
